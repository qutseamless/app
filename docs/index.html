    Getting Started

    this section describes common scripts that can be used to get started.

    • continuously build and serve: npm start.
    • continuously test: npm test.
    • continuously document: npm run document.
    • update dependancies: npm i.

    Source Structure

    this section describes the structure we will be using.

    main.jsx

    • This is where react will be initialized and mounted to the DOM.

    logic

    • Each folder in logic will encompass its own: actions, core, and reducer.

    libs

    • Any business heavy logic should be lifted from the logic and refactored in libs as to keep code clean.

    styles (STRICTLY FOR NONGENERATIVE STYLES)

    • Style mix-ins, variables and placeholder classes belong here. this is to avoid multiple imports of the same styles.

    Testing

    testing will be conducted using the karma test runner, mocha, and chrome. specs will be placed in the same directory as the code they are testing and will be noted as '/spec.jsx?/'
